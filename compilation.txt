To compile the gameEditor and wxGameEditor projects (editor version, not the stand alone engines):

1) Get the RakNet at http://www.jenkinssoftware.com/

Configure the LibStatic project to:
Multi-threaded Debug

Build the LibStatic version of the RakNet

2) Get the LLVM and clang (you need to get the last version via svn)

cd llvm
svn co http://llvm.org/svn/llvm-project/llvm/trunk .

cd tools
svn co http://llvm.org/svn/llvm-project/cfe/trunk clang 


Put the follow llvm dirs in your include path:

llvm/tools/clang/include
llvm/include
llvm/win32

Put the follow llvm dirs in your library path:

llvm/win32

Configure all projects of the LLVM to:
Multi-threaded Debug

3) Get the wxWidgets at http://www.wxwidgets.org/

Compile with the option Multi-threaded (/MT or /MTd)

Put the follow llvm dirs in your include path:

wxWidgets/include
wxWidgets/include/msvc
wxWidgets/contrib/include
wxWidgets/contrib/wxstedit/include

Put the follow llvm dirs in your library path:

wxWidgets/lib/vc_lib

Configure all projects of the wxWidgets to:
Multi-threaded Debug

Windows:

In the Visual Studio 2008, choose the version you want to compile:

	- View->Property Manager
	- In gameEditor project , _BaseConfiguration of any configuration, in the User Macros put the configuration you want in the VersionType:

- To compile for the GP2X select 'Windows Mobile 5.0 Pocket PC SDK (ARMV4I)' (you need to install the Windows Mobile 5.0 SDK for Pocket PC and have the devkitpro installed (http://www.devkitpro.org/))
		
	- Para compilar em outra plataforma, se necessario:
	
		- Ativar 'GCC Compiler' em Project -> 'Custom Build Rules' nos projetos: 
		
			eiclib, engine, gameEditor, gameEngine, mikmod, native_midi, RakNet, 
			staticSDL, staticSDL_mixer, timidity
			
		- Em Project -> 'Tool Build Order' Mover 'GCC Linker' e 'GCC Archiver' para depois de 'Linker Tool'
		- Colocar as bibliotecas geradas em Properties -> 'GCC Linker' -> General -> 'Additional Dependencies'
			
			../output/PocketPC/RakNet.lib;../output/PocketPC/gameEngine.lib;../output/PocketPC/engine.lib;../output/PocketPC/eiclib.lib;../output/PocketPC/staticSDL_mixer.lib;../output/PocketPC/mikmod.lib;../output/PocketPC/timidity.lib;../output/PocketPC/native_midi.lib;../output/PocketPC/staticSDL.lib;E:\devkitpro\devkitGP2X\sysroot\usr\lib\libm.a;E:\devkitpro\devkitGP2X\sysroot\usr\lib\libpthread.a;E:\devkitpro\devkitGP2X\arm-linux\lib\libstdc++.a;$(NOINHERIT)
		

- Use the GameControl::CreateEngines() function to create the bin files from the compiled executables:
  transforma os arquivos .exe em .bin compactados:
  
	gameEditorWindows.exe		-> bin\\windows.bin
	gameEditorPocket.exe		-> bin\\pocketpc.bin
	gameEditorLinux				-> bin\\linux.bin
	gameEditorPocket.exe.gpe	-> bin\\gp2x.bin


- To make a Pocket PC compilation works in the Pocket PC 2002, uses: /subsystem:windowsce,3.00 /MACHINE:ARM
