The Include Directive

A preprocessor directive of the form

#include 4#4file-name5#5

causes the above line to be replaced by the entire contents of the file file-name. The file is searched for according to the standard search path list (see also discussion on adding paths to the search list on page ). A preprocessor directive of the form

#include ``file-name''

causes the search to begin first in the current working directory and if this fails it searches for the file in those directories specified via the search path list.

Alternatively, a preprocessor directive of the form

#include token-sequence

causes first the token-sequence to be expanded as is for normal identifier text and strictly ISO style, one of the two forms, 6#6 or "7#7", should result. However, EiC first expands the token-sequence and then treats the resulting text as a file-name; that is, a preprocessor directive of the form

#include file-name
is legal in EiC.
