Module names and assumptions

The name given to a module, module_name, is important for several reasons, and particularly so for builtin modules. This is because on startup, EiC will expect to call each builtin-modules initialising function `module_module_name' (see also pg: ). This call will be automatically setup by EiC's makefile system, using the file `EiC/module/modules.calls'. Entries into this file will de done via each Makefile installed in each builtin module. The entry for module directory `XXXX' would look like: 

        #ifndef NO_module_XXXX
        module_XXXX();
        #endif

The file `EiC/module/modules.calls' is inturn included into EiC's main function.
